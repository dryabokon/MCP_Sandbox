# explain-agent

Read the agent file at `agents/$ARGUMENTS.py` and its system prompt at `agents/$ARGUMENTS.md`.

Explain in plain language:
1. What this agent does
2. What tools it has access to and why
3. What kinds of user requests it can handle
4. Any behavioral rules or constraints defined in the `.md`

Be concise. Use bullet points. Target audience is a developer seeing this agent for the first time.
